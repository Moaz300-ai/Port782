# دليل رفع موقعك على GitHub Pages

هذا الدليل يشرح خطوات رفع موقعك على GitHub واستضافته مجاناً باستخدام GitHub Pages.

## المتطلبات الأساسية

1. حساب على [GitHub](https://github.com)
2. تثبيت [Git](https://git-scm.com/downloads) على جهازك
3. معرفة أساسية بأوامر Git (سنشرح الأوامر الأساسية في هذا الدليل)

## الخطوات التفصيلية

### 1. إنشاء مستودع جديد على GitHub

1. قم بتسجيل الدخول إلى حسابك على [GitHub](https://github.com)
2. انقر على زر "+" في أعلى الصفحة ثم اختر "New repository"
3. أدخل اسم المستودع (يفضل استخدام `username.github.io` حيث `username` هو اسم المستخدم الخاص بك)
4. اختر "Public" كنوع للمستودع
5. لا تقم بتحديد خيار "Initialize this repository with a README"
6. انقر على "Create repository"

### 2. تهيئة Git محلياً ورفع الملفات

افتح Terminal (على macOS/Linux) أو Command Prompt/Git Bash (على Windows) ونفذ الأوامر التالية:

```bash
# انتقل إلى المجلد الذي يحتوي على ملفات موقعك
cd /path/to/your/website/files

# قم بتهيئة مستودع Git محلي
git init

# أضف جميع الملفات إلى منطقة التحضير
git add .

# قم بعمل commit للتغييرات
git commit -m "النسخة الأولى من موقع الألغاز والأسئلة"

# قم بربط المستودع المحلي بالمستودع البعيد على GitHub
# استبدل USERNAME باسم المستخدم الخاص بك وREPO_NAME باسم المستودع
git remote add origin https://github.com/USERNAME/REPO_NAME.git

# ادفع التغييرات إلى المستودع البعيد
git push -u origin master
```

### 3. تفعيل GitHub Pages

1. انتقل إلى صفحة المستودع على GitHub
2. انقر على "Settings" في شريط التنقل العلوي
3. انتقل إلى قسم "Pages" في القائمة الجانبية
4. في قسم "Source"، اختر "master" (أو "main") من القائمة المنسدلة
5. انقر على "Save"
6. انتظر بضع دقائق حتى يتم نشر موقعك
7. ستظهر رسالة تؤكد نشر موقعك وتعرض الرابط (عادة ما يكون `https://username.github.io`)

### 4. التحقق من موقعك

1. انتقل إلى الرابط الذي تم توفيره في الخطوة السابقة (`https://username.github.io`)
2. تأكد من أن موقعك يعمل بشكل صحيح
3. إذا واجهت أي مشاكل، تحقق من سجل الأخطاء في قسم "Actions" في مستودعك على GitHub

## ملاحظات هامة

- تأكد من وجود ملف `index.html` في المجلد الرئيسي للمستودع
- ملف `.nojekyll` يمنع GitHub من معالجة موقعك باستخدام Jekyll
- ملف `_config.yml` يحدد إعدادات GitHub Pages مثل السمة واللغة
- قد يستغرق نشر موقعك بضع دقائق بعد دفع التغييرات

## استخدام نطاق مخصص (اختياري)

إذا كنت ترغب في استخدام نطاق مخصص بدلاً من `username.github.io`:

1. قم بشراء نطاق من أي مزود خدمة تسجيل النطاقات
2. في صفحة مستودعك، انتقل إلى "Settings" ثم "Pages"
3. في قسم "Custom domain"، أدخل النطاق الخاص بك
4. انقر على "Save"
5. قم بإنشاء ملف `CNAME` في المجلد الرئيسي للمستودع يحتوي على اسم النطاق الخاص بك
6. قم بإضافة سجلات DNS التالية في إعدادات النطاق لدى مزود الخدمة:
   - سجل A يشير إلى عناوين IP التالية:
     - 185.199.108.153
     - 185.199.109.153
     - 185.199.110.153
     - 185.199.111.153
   - أو سجل CNAME يشير إلى `username.github.io`

## تحديث موقعك

لتحديث موقعك في المستقبل، قم بتنفيذ الأوامر التالية:

```bash
# قم بإجراء التغييرات على ملفات موقعك

# أضف التغييرات إلى منطقة التحضير
git add .

# قم بعمل commit للتغييرات
git commit -m "وصف التغييرات التي قمت بها"

# ادفع التغييرات إلى المستودع البعيد
git push
```

## استكشاف الأخطاء وإصلاحها

- **الموقع لا يظهر**: تأكد من وجود ملف `index.html` في المجلد الرئيسي وانتظر بضع دقائق
- **مشاكل في التنسيق**: تأكد من أن جميع المسارات نسبية وليست مطلقة
- **أخطاء في الصور**: تأكد من أن جميع الصور موجودة في المستودع وأن المسارات صحيحة
- **مشاكل في JavaScript**: افتح وحدة تحكم المتصفح للتحقق من وجود أخطاء
